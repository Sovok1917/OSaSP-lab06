
# README.txt

## Project: Multi-threaded Index File Sorter

This project consists of three programs:
1.  `gen`: Generates a binary index file with a specified number of records.
2.  `sort_index`: Sorts the index file using multiple threads and memory-mapped I/O.
3.  `view`: Displays the contents of an index file to standard output.

The index file format consists of a header (`struct index_hdr_s`) followed by a series of index records (`struct index_s`). Each record contains a timestamp (`double time_mark`) and a record number (`uint64_t recno`).

Sort an index file:
The sort_index program takes memory size, number of blocks, number of threads, and the filename.

    memsize: Working buffer size, must be a multiple of system page size.

    blocks: Number of blocks to divide memsize into; must be a power of 2.

    threads: Number of threads to use.

    filename: The file to sort (generated by gen).

Important: For sort_index to work as specified by the problem (simplifying chunk logic), the total data size of the records in the file (num_records * sizeof(struct index_s)) must be an exact multiple of memsize. You may need to choose num_records for gen and memsize for sort_index carefully to satisfy this. sort_index will verify this condition.


./build/sort_index <memsize> <blocks> <threads> <filename>

Example (assuming page size is 4096):
Let's say memsize is 81920 (20 pages).
sizeof(struct index_s) is 16 bytes.
Number of records in one memsize chunk: 81920 / 16 = 5120.
If gen created a file with num_records = 2 * 5120 = 10240 (which is a multiple of 256), then this file's data size (10240 * 16 = 163840) is a multiple of memsize (81920).


# First, determine system page size:
getconf PAGESIZE
# Let's assume it's 4096
# memsize = 20 * 4096 = 81920
# blocks = 32 (power of 2)
# threads = 4
# (Check: blocks >= 4*threads => 32 >= 4*4 => 32 >= 16, OK)
# (Check: k <= threads <= 8k. If k=4, 4 <= 4 <= 32, OK)

# Generate file: num_records = (2 * memsize) / sizeof(index_s) = (2 * 81920) / 16 = 10240
# 10240 is a multiple of 256 (10240 / 256 = 40)
./build/gen 10240 my_index_large.dat

# Sort the file
./build/sort_index 81920 32 4 my_index_large.dat

Notes

    The sort_index program requires pthread library and math library (-lm). The Makefile handles this.

    The MJD (Modified Julian Date) for timestamps in gen is calculated based on a range from 1900-01-01 up to "yesterday" relative to the system's current UTC time.
